# Progress

A progress bar is a graphical control element used to visualize the progression of an extended computer operation, such as a download, file transfer, or installation.

## Features

- Keyboard support for actions.
- Support for hover, focus and active states.

## Installation

```jsx
yarn add @universa11y/progress
```

## Usage

```jsx
import { createProgress } from '@universa11y/Progress';
const Progress = createProgress({
  Root,
  FilledTrack,
});
```

These are the styled components that we need to pass in the creator API:

### Root

```jsx
const Root = styled(
  View,
  {
    ...
  },
);
```

### FilledTrack

```jsx
const Root = styled(
  View,
  {
    ...
  },
);
```

## Anatomy

```jsx
export default () => (
  <Progress>
    <Progress.FilledTrack></Progress.FilledTrack>
  </Progress>
);
```

## API Reference

### Progress

Contains all Progress related layout style props and actions.
It inherits all the properties of React Native's [Progress](https://reactnative.dev/docs/Progress#props) component.

## Accessibility

### Keyboard

- Adheres to the progressbar [role requirements]("https://www.w3.org/TR/wai-aria-1.2/#progressbar").

- On web, aria-value now, aria-value min and aria-value max is used to ensure the progress percent is visible to screen readers.

- On mobile, [accessibilityValue]('https://reactnative.dev/docs/accessibility#accessibilityvalue') is used to ensure it is announced by Talkback and VoiceOver.
