# Switch

The Switch provides an alternative to the Checkbox component. It also allows a user to switch between enabled or disabled states.

<AppProvider>
  <CodePreview
    showComponentRenderer={true}
    showArgsController={false}
    metaData={{
      code: ` 
      function App () {
  const [isEnabled, setIsEnabled] = useState(false);
        return (
          <Wrapper>
       <Switch
        value={isEnabled}
        onValueChange={(val) => setIsEnabled(val)}
      />
          </Wrapper>
    )
    }`,
      scope: { Switch: AccessibleSwitch, Wrapper, useState },
      argsType: {},
    }}
  />
</AppProvider>

<br />

## Features

- Keyboard support for actions.
- Support for native switch.

## Installation

```jsx
yarn add @universa11y/switch
```

## Usage

```jsx
import { createSwitch } from '@universa11y/switch';
const Switch = createSwitch({
  Root,
});
```

These are the styled components that we need to pass in the creator API:

### Root

```jsx
const Root = styled(
  Pressable,
  {
    ...
  },
);
```

## Anatomy

```jsx
export default () => <Switch />;
```

## API Reference

### Switch

Contains all button related layout style props and actions.
It inherits all the properties of React Native's [Switch](https://reactnative.dev/docs/switch#props) component.

## Accessibility

- On mobile, uses native switch which is fully accessible.
- On web, it uses checkbox with a role set to switch.

## Dependencies

- `"@react-stately/toggle"`
- `@react-native-aria/focus`
- `@react-native-aria/interactions`
- `react-native-svg`
