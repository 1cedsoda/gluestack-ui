# Theme Tokens

- Dank uses "theme tokens" to store design-related values such as colors, typography, spacing, and more. Theme tokens are a way of centralizing these values in a single location, making it easy to change and maintain the design of your application.
- To use theme tokens, you need to create a theme object that contains all the design tokens you want to use in your application. Here is an example of a theme object:

```js
const theme = {
  colors: {
    primary: 'blue',
    secondary: 'green',
    text: 'black',
    background: 'white',
  },
  fonts: {
    body: 'Arial, sans-serif',
    heading: 'Georgia, serif',
  },
  fontSizes: {
    small: '12px',
    medium: '16px',
    large: '20px',
  },
  space: [0, 4, 8, 16, 32],
  breakpoints: {
    xs: 0,
    sm: 600,
    md: 960,
    lg: 1280,
    xl: 1920,
  },
};
```

In this example, the theme object contains different properties for colors, fonts, font sizes and spacing. You can add as many properties as you need, such as typography, border radius, shadows, etc.

## Using Aliases

Dank supports the use of "aliases" to make it easier to work with theme tokens and other configuration options. Aliases allow you to give a shorthand name to a theme token or configuration value, making it easier to reference them in your code.

To use aliases, you need to define them in your config file, such as **`dank.config.js`** or in your theme object. Here is an example of how you can define aliases in a config file:

```js
export const config = {
  aliases: {
    bg: 'backgroundColor',
    p: 'padding',
    m: 'margin',
    c: 'color',
    f: 'fontSize',
  },
  theme: {
    colors: {
      primary: 'blue',
      secondary: 'green',
      text: 'black',
      background: 'white',
    },
    fonts: {
      body: 'Arial, sans-serif',
      heading: 'Georgia, serif',
    },
    fontSizes: {
      small: '12px',
      medium: '16px',
      large: '20px',
    },
    space: [0, 4, 8, 16, 32],
    breakpoints: {
      xs: 0,
      sm: 600,
      md: 960,
      lg: 1280,
      xl: 1920,
    },
  },
};
```

In this example, we have defined aliases for **`backgroundColor`**, **`padding`**, **`margin`**, **`color`**, and **`fontSize`**. Once the aliases are defined, you can use them in your component styles instead of the full property name.
